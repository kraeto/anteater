What is included in the Quality Assurance track?

  1. Education about Software Testing & Quality Assurance
    • Supplemented Reading: "Software Testing (Second Edition)" by Ron Patton
    • The Software Development Life Cycle
    • Static and dynamic testing:
      • Static testing: Testing the software while it is powered down, or testing the Functional Specification Document, or FSD
      • Dynamic testing: Testing the software while it is powered on, or carefully comparing the FSD to the software
    • White-box and black-box testing
      • White-box testing: Testing the software with knowledge of the code
      • Black-box testing: Testing the software without knowledge of the code
    • Stress testing and load testing
      • Stress testing: Running the software with very low resources to determine what the software does post-stress
      • Load testing: Overloading the software with too much input, connections, or users
  2. Learning about bugs
    • The importance of Quality Assurance inside and outside of software development
    • Where are bugs most commonly found during the Software Development Life Cycle?
    • Case studies on actual bugs, such as the Y2K Bug, which was one of the infamous computer bug scares that reset the date back to 1900 instead of 2000.
